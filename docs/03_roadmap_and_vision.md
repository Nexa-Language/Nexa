# Nexa 路线图：从 MVP 到 AVM (Agent Virtual Machine)

Nexa 语言的演进路线严格遵循 "Think Big, Start Small" 的原则。我们的终极目标是创建一个原生且安全的智能体执行引擎。

## 阶段 1-4: v0.1 到 v0.8.x (已完成 ✅)
覆盖了 MVP 转译器、管道操作、架构重构、安全模块 (.nxs/.nxlib)、契约式编程 (Protocols)、多模型路由、持久化记忆及 Markdown 动态工具体系编排。

## 阶段 5: v0.9 认知与治理增强 (Cognitive & Governance)
* **双系统思维 (System 1 & System 2)：** 原生支持 `reflect { ... } until condition` 等语法。
* **状态机约束 (FSM Guardrails)：** 强制 Agent 状态转移路径，杜绝死循环。
* **事件驱动与跨平台交互：** `whenever event_name -> action` 异步事件监听。

## 阶段 6: v1.0 全景路线图确立 (The Ultimate AVM) 🚀
v1.0 版本将着重打造完整的 AI 原生体系，核心包含以下 6 个全新架构维度设计规划：

### 1. Memory/Context (上下文与长驻记忆体系)
统一支持 `local`, `shared`, `persistent` 的分级缓存控制机制。在 v1.0 中将引入向量数据库内建接入或原生 RAG 挂载点，提供透明的自动上下文压缩 (Auto-summarization) 与海量会话找回能力。

### 2. Resilience/Fallback (容错与灾备降级)
提出原生的 `fallback` 关键字层。Agentic Workflow 中经常面临 API 宕机、解析失败或死循环。通过 `action_a fallback action_b` 原语，使流程在发生 `Exception` 时，无缝安全地退化到备用流程或提示降级。

### 3. Observability/Tracing (执行观测与时空穿梭)
抛弃粗糙的终端抓取，引入系统级的 AST 执行切面 (AOP)。将每一步 Agent 思考流、工具调用与上下文变更进行结构化打点 (Log/Trace)。支持 "Time-travel Debugging"，允许开发者在沙盒里回滚断点，从特定的历史树节点重新启发推理。

### 4. Event-driven/Async (异步事件总线)
支持 `emit` 和 `on` 关键字。当爬虫工具发现新数据，或者外部 Webhook 到达时，通过异步消息泵直接唤醒沉睡的 Agent 进行并行消化，脱离单调的线性 `flow` 束缚。

### 5. Multi-Modal (多模态感知内化)
不再将视觉、音频视为二等公民。引入内建的视觉处理语法如 `img("some_file.jpg")`，并将在编译器层面自动拦截并展开成多模态 LLM 能够识别的 Base64 Payload。为具备"世界感知"能力的机器人奠定基础设施。

### 6. MCP Ecosystem (工具生态自治)
全面整合 Model Context Protocol。引入 `nxpm` 甚至直接复用生态中海量的只读/可写 Server 集成。让 Nexa 代码里一个简短的 `uses mcp:"github"` 或者 `uses mcp:"sqlite"` 就能将全世界的服务变成函数的本地执行端点点。



## 总设计师的未来畅想

### 功能上（已经做过的打钩）

1. 让流运算符 `>>` 支持更复杂的拓扑结构，如分叉、合流等，真正实现复杂 Agent DAG 的无缝编排。
2. 引入原生的错误处理机制，如 `try/catch` 块，允许开发者优雅地处理运行时异常和错误。
3. 原生支持声明 `uses mcp:"..."` 来引入外部服务，形成一个真正的工具生态系统。
4. Rust AVM，改变现有脚本式的解释执行模式，转向编译型的 AVM，提升性能和安全性。
5. 引入wasm沙盒化执行tools，提供更强的安全隔离和跨平台兼容性。
6. 设计一个可视化的 Agent DAG 编辑器，允许开发者通过拖拽的方式构建和调整 Agent 的协作流程。
7. 在AVM层面引入智能调度器，基于当前系统负载、Agent的优先级和资源需求，动态调整Agent的执行顺序和资源分配。
8. 在AVM层面接管内存上下文，使用类似向量虚存分页的机制，自动管理Agent的上下文数据，提供透明的上下文切换和持久化能力。
9. 提供test子类，可以进行各种测试，比如测试agent连通性、性能测试、压力测试等，帮助开发者确保Agent的稳定性和可靠性。
10. 提供运行时根据上下文动态生成新agent或者改变模型配置的方法，增强系统的适应性和灵活性。
11. 引入基于角色的访问控制 (RBAC)，允许开发者定义不同角色的权限，确保系统的安全性和数据隐私。
12. 接入open-cli，让agent可以通过命令行工具与外部系统进行交互，扩展agent的应用场景和功能。`https://github.com/spectreconsole/open-cli`
13. 提供宿主互操作性（Python Interop），真实业务中，开发者通常需要将 Agent 接入现有的 FastAPI、Django 等后端。应当提供一个 Python SDK 入口，例如：
```python
from nexa import NexaRuntime
nexa = NexaRuntime()
result = nexa.run_agent("my_agent.nxs", input_data={"key": "value"})
```
14. 强化 semantic_if 的性能，引入基于启发式的快速路径评估机制，在满足条件的情况下直接跳过大模型调用，提升条件判断的效率和响应速度。

### 开发者体验

1. **IDE 集成**：提供官方的 VSCode 插件，支持语法高亮、智能提示、错误检查和一键运行。
    - 语法高亮，基于现有EBNF规则，编写TextMate语法定义文件.tmLanguage.json，重点高亮关键字、变量和函数调用。
    - 语言服务器提示，基于LSP协议实现，实时运行parser进行语法分析，提供智能提示和错误检查，红色波浪线标记语法错误。
    - 代码补全，基于上下文分析和预定义的函数/变量列表，提供智能代码补全建议。
2. **包管理器**：NxPM雏形
    - 利用现有的.nxs/.nxlib格式，设计一个简单的包管理器，支持安装、更新和发布Nexa库。
    - 提供命令行工具 `nxpm`，支持 `nxpm install <package>` 和 `nxpm publish` 等基本功能。
    - 基于Github Registry或类似平台，搭建一个公共的Nexa包仓库，鼓励社区贡献和共享工具库。
3. **Source Map源码映射**：错误回溯

    - 在运行时捕获异常时，利用源代码映射文件将Python异常回溯到对应的Nexa代码行，提供更友好的错误提示和调试体验。
4. **调试工具**：提供一个交互式的调试工具，允许开发者在运行时查看Agent的状态、变量值和执行流程，支持断点设置和单步执行。

    - 在调试模式下，提供实时的Agent状态和变量值输出，帮助开发者理解程序的运行情况和定位问题。
