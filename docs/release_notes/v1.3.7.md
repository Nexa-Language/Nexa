# Nexa v1.3.7 Release Notes

**发布日期**: 2026-04-23  
**主题**: Language Expressiveness — Pipe Operator, Defer, Null Coalescing, String Interpolation, Pattern Matching, ADT

## 概述

Nexa v1.3.7 发布了 6 个语言表达力增强特性，大幅提升了 Nexa 的语法灵活性和编程范式支持。

## 新特性

### P3-2: Pipe Operator `|>` — 84 tests (Batch A)

左到右数据流：`x |> f` 脱糖为 `f(x)`，`x |> f(a,b)` 脱糖为 `f(x, a, b)`。

```nexa
result |> format_output
data |> std.text.upper
prompt |> agent.run |> extract_answer
```

修改文件：`src/nexa_parser.py` (pipe_expr), `src/ast_transformer.py` (pipe_chain_expr)

### P3-5: Defer Statement — 84 tests (Batch A)

确保清理操作在作用域退出时执行（LIFO 顺序），即使发生错误。

```nexa
defer cleanup(db)
defer log("operation complete")
defer agent_cleanup(agent)
```

修改文件：`src/nexa_parser.py`, `src/ast_transformer.py`, `src/code_generator.py` (_nexa_defer_stack/_nexa_defer_execute), `src/runtime/__init__.py`

### P3-6: Null Coalescing `??` — 84 tests (Batch A)

安全回退值：`expr ?? fallback` 在 expr 为 None/Option::None/空 dict 时返回 fallback。

```nexa
result ?? "fallback"
config.timeout ?? 30
agent.run(prompt) ?? "I couldn't process that"
```

修改文件：`src/nexa_parser.py`, `src/ast_transformer.py`, `src/code_generator.py` (_nexa_null_coalesce), `src/runtime/__init__.py`

### P3-1: String Interpolation `#{expr}` — 100 tests

Ruby 风格 `#{expr}` 语法在双引号字符串中插值。支持标识符、点访问、括号访问。

```nexa
"Hello #{name}, you are #{age} years old!"
"Status: #{result ?? 'pending'}"
"Agent #{agent.name} responding"
```

修改文件：`src/ast_transformer.py` (_parse_string_interpolation), `src/code_generator.py` (_generate_interpolated_string), `src/runtime/__init__.py` (_nexa_interp_str), `src/runtime/stdlib.py` (std.string.interpolate), `src/runtime/inspector.py`, `src/runtime/type_system.py`

### P3-3: Pattern Matching + Destructuring — 91 tests

7 种模式类型：Wildcard `_`、Variable、Literal、Tuple `(a,b)`、Array `[a,b,..rest]`、Map `{name,age:a,..other}`、Variant `Option::Some(v)`。

```nexa
match result {
    Option::Some(answer) => answer
    Option::None => "no response"
}

let (key, value) = entry
for (name, score) in rankings { ... }
```

新增文件：`src/runtime/pattern_matching.py`
修改文件：`src/nexa_parser.py` (match_expr + 7 pattern rules), `src/ast_transformer.py` (18 handlers), `src/code_generator.py`, `src/runtime/__init__.py`, `src/runtime/stdlib.py` (std.match)

### P3-4: ADT (Struct/Trait/Enum) — 100 tests

代数数据类型：struct、enum、trait、impl 声明，handle-as-dict 模式，ContractViolation 集成。

```nexa
struct Point { x: Int, y: Int }
enum Option { Some(value), None }
enum Result { Ok(value), Err(error) }
trait Printable { fn format() -> String }
impl Printable for Point { fn format() -> String { ... } }
```

新增文件：`src/runtime/adt.py`
修改文件：`src/nexa_parser.py` (struct_decl/enum_decl/trait_decl/impl_decl), `src/ast_transformer.py` (10 handlers), `src/code_generator.py` (ADT imports + 5 generate methods), `src/runtime/__init__.py`, `src/runtime/stdlib.py` (std.struct/std.enum/std.trait)
