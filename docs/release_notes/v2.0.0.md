# Nexa v2.0.0 Release Notes

**Release Date**: 2026-05-15

## 🎯 Summary

Nexa v2.0 introduces the **Harness Native Runtime** — transforming the Harness six-tuple H=(E,T,C,S,L,V) from compile-time validation into runtime first-class primitives. This is the most significant architectural evolution since the v1.0 AVM introduction.

## 🆕 New Features

### New Syntax Primitives

```nexa
// E-dimension: autoloop
autoloop max_steps: 10, exit_when: "done", timeout: 300 { ... }

// T-dimension: @tool annotation
@tool("description") fn my_tool(x: string): string { ... }

// C-dimension: with_context
with_context max_tokens: 100000, strategy: importance_weighted { ... }

// S-dimension: snapshot/restore
snap = snapshot();
restore(snap);

// L-dimension: lifecycle hooks + reflect
before_step { ... }
after_step { ... }
reflect "Consider potential side effects";

// V-dimension: verify
verify result satisfies ExpectedOutput;

// Actor: spawn/pass/await
reviewer = spawn CodeReviewer("review code");
pass "review these changes" to reviewer;
result = await reviewer;
```

## 🔧 Bug Fixes & Improvements

### AVM Rust Backend

- Fixed all 17 compilation errors in `avm/src/runtime/jobs.rs`:
  - `self.now_secs()` → `Self::now_secs()` (associated function, not method)
  - `Option<f64>.cmp()` → `unwrap_or(0.0).total_cmp()` (f64 comparison)
  - `String/&str` type mismatches → `.map(|s| s.as_str())` pattern
  - Borrow checker conflicts → pre-clone before move
- Fixed all 21 warnings across AVM codebase:
  - Removed unused imports (8 files)
  - Prefixed unused variables with `_` (7 occurrences)
  - Added `#[allow(dead_code)]` for struct fields/methods (4 occurrences)

## 🔄 Breaking Changes

- `nexa build` now accepts `--harness=warn|error|off` flag
- Agent declarations now support Harness dimension annotations
- `autoloop`, `with_context`, `snapshot/restore`, `verify`, `reflect` are new keywords

## 📈 Migration Guide

v1.x code is fully backward compatible. v2.0 features are opt-in:

```nexa
// v1.x code still works unchanged
agent ChatBot { prompt: "..." }
flow main { result = ChatBot.run("hello") }

// v2.0: add Harness primitives incrementally
flow main {
    autoloop max_steps: 5 {
        with_context max_tokens: 50000 {
            try_agent {
                result = ChatBot.run("hello")
                verify result satisfies string
            } catch_correction(e: ToolError) {
                reflect "Adjusting approach"
            }
        }
    }
}
