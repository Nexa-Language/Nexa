# Nexa v1.1.0 Release Notes

**发布日期**: 2026-04-23  
**主题**: Intent-Driven Development (IDD)

## 概述

Nexa v1.1.0 引入了 Intent-Driven Development (IDD) 系统——Nexa 的核心差异化特性。IDD 让需求文档变成可执行测试，形成"需求→实现→验证"的闭环，融合了 Agent-native 验证能力。

## 新特性

### P0-1: Intent-Driven Development (IDD) — 104 tests

Nexa 的 IDD 系统提供了一种结构化方式来定义、追踪和验证智能体行为意图。开发者可以声明式地指定智能体 *应该做什么*，然后根据这些规范验证实现正确性。

**核心组件**：

1. **IAL (Intent Assertion Language)** — 术语重写引擎，将自然语言断言递归解析为可执行测试
2. **`.nxintent` 文件** — YAML 格式需求文档，定义 Feature/Scenario/Glossary
3. **`@implements` 注解** — 将代码链接到需求，形成需求→代码→验证的闭环

**新增文件**：

| 文件 | 说明 |
|------|------|
| `src/ial/primitives.py` | Intent 原语定义 (action, goal, constraint, preference) |
| `src/ial/vocabulary.py` | Intent 术语词汇管理 |
| `src/ial/standard.py` | 标准术语词汇 (40+ 内置 intents) |
| `src/ial/resolve.py` | Intent 解析引擎 (模糊匹配、语义分组) |
| `src/ial/execute.py` | Intent 执行和验证引擎 |
| `src/ial/__init__.py` | 模块导出 |

**CLI 命令**：

```bash
nexa intent-check    # 验证代码是否符合 intent
nexa intent-coverage # 显示特性覆盖率百分比
nexa inspect         # 结构化分析 agents/tools/flows
```

**语法示例**：

```nexa
// @implements 注解链接 intent
// @implements: feature.weather_bot
agent WeatherBot uses WeatherAPI implements WeatherReport {
    role: "Weather Assistant",
    prompt: "Provide weather information"
}
```

**`.nxintent` 文件格式**：

```yaml
Feature: Weather Bot
  id: feature.weather_bot
  description: "Weather information agent"

  Scenario: Weather query
    When a user asks "What is the weather in Beijing?"
    → the agent responds with "weather"
    → the response is valid
```

**关键能力**：
- Intent 声明：`intent Bot shall "respond politely to user questions"`
- Intent 覆盖率测量：代码路径被 intents 覆盖的百分比
- Intent 验证：运行时检查 intent 满足情况
- 模糊 Intent 匹配：部分 intent 规范的近似匹配解析
