# Nexa v1.2.0 Release Notes

**发布日期**: 2026-04-23  
**主题**: Design by Contract (契约式编程)

## 概述

Nexa v1.2.0 引入了 Design by Contract (DbC) 系统，将契约式编程范式带到 Agent 编程领域。开发者可以指定 `requires` (前置条件)、`ensures` (后置条件) 和 `invariant` (不变条件) 来在运行时验证 Agent 行为。

## 新特性

### P0-2: Design by Contract — 47 tests

Nexa 的 Contract 系统提供运行时行为验证，确保 Agent 的输入满足前置条件、输出满足后置条件、状态始终满足不变条件。

**Contract 语法**：

```nexa
requires: input != None
ensures: result.length > 0
invariant: self.state in ["idle", "running"]
```

**新增文件**：

| 文件 | 说明 |
|------|------|
| `src/runtime/contracts.py` | ContractSpec, ContractClause, ContractViolation, check_requires, check_ensures, capture_old_values |

**ContractViolation 集成**：
- HTTP Server: 401 → requires violation, 403 → ensures violation
- KV Store: 失败操作 → ensures violation
- Concurrent: 任务失败 → contract violation
- ADT: 无效操作 → ContractViolation

**语法示例**：

```nexa
agent SecureBot {
    role: "Secure Agent",
    prompt: "Process requests safely"
    
    requires: user_input != None and user_input.length < 1000
    ensures: response contains "processed"
    invariant: auth_level >= 1
}
```

**关键概念**：
- `requires` — 前置条件：函数/方法调用前必须满足的条件
- `ensures` — 后置条件：函数/方法返回后必须满足的条件  
- `invariant` — 不变条件：对象生命周期中始终满足的条件
- `ContractViolation` — 契约违反时抛出的异常，可被 `try/catch` 捕获
