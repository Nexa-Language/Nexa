# Nexa v1.3.0 Release Notes

**发布日期**: 2026-04-23  
**主题**: Agent-Native Tooling + Compiler Architecture Overhaul

## 概述

Nexa v1.3.0 引入了 Agent-Native CLI 工具系统，并完成了编译器架构文档的全面重写，反映从 v0.9-alpha 到 v1.3.0 的架构演进。

## 新特性

### P0-3: Agent-Native Tooling — 41 tests

Nexa 的内置 CLI 工具提供智能体感知的代码检查、验证和 lint 功能：

**CLI 命令**：

| 命令 | 功能 |
|------|------|
| `nexa inspect` | 结构化分析 agents、tools、flows |
| `nexa validate` | 语义验证 Nexa 程序 |
| `nexa lint` | 风格和最佳实践检查 |
| `nexa intent-check` | Intent 覆盖率验证 |
| `nexa intent-coverage` | 覆盖率百分比报告 |

**新增文件**：

| 文件 | 说明 |
|------|------|
| `src/runtime/inspector.py` | 结构化分析引擎 |
| `src/runtime/validator.py` | 语义验证引擎 |

**Agent-Native 特性**：
- `inspect` 输出包含 agent 工具链和依赖关系图
- `validate` 检查 agent 的 prompt/model/工具是否配置完整
- `lint` 检查 agent 声明是否符合最佳实践（prompt 长度、命名规范等）

## 编译器架构重写

`docs/02_compiler_architecture.md` 从 v0.9-alpha 版本全面重写为 v1.3，新增以下章节：

1. **AST Transformer Scoring System** — `_score_ast_node()` 方法解决 Lark Earley 解析器歧义问题
2. **BOILERPLATE Code Generation Pattern** — 每个功能模块的导入和辅助函数模板
3. **Handle-as-dict Pattern** — 所有运行时 handle 使用 `_nexa_*` 前缀键的 Python dict 表示
4. **Thread-safe Registry Pattern** — `_registry_lock` + `_id_counter` 的全局注册表模式
5. **StdTool Namespace Pattern** — `std.*` 命名空间通过 StdTool 在 stdlib.py 注册
