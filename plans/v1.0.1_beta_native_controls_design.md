# Nexa v1.0.1-beta 架构设计文档

## 设计目标

1. 补齐高频基础控制流 (if/else if/else, for each, while, break, continue)
2. 引入无渲染冲突的 Python 逃生舱
3. 确保与现有 `semantic_if` 和 `loop...until` 的清晰区分

## 一、传统控制流语法设计

### 1.1 if/else if/else 语句

**设计原则**: 与 `semantic_if` 明确区分，使用传统编程语言的语法风格。

**EBNF 语法设计**:
```ebnf
// 传统 if/else if/else - 确定性条件分支
traditional_if_stmt: "if" "(" traditional_condition ")" block else_if_clause* else_clause?
else_if_clause: "else" "if" "(" traditional_condition ")" block
else_clause: "else" block

// 条件表达式支持逻辑运算符
traditional_condition: logical_expr
logical_expr: comparison_expr (("and" | "or") comparison_expr)*
comparison_expr: expression comparison_op expression
               | expression  // 简单布尔判断
```

**示例代码**:
```nexa
flow example() {
    score = 85
    
    // 传统 if/else if/else
    if (score >= 90) {
        grade = "A"
    } else if (score >= 80) {
        grade = "B"
    } else if (score >= 70) {
        grade = "C"
    } else {
        grade = "F"
    }
}
```

### 1.2 for each 循环

**EBNF 语法设计**:
```ebnf
// for each 遍历数组/集合
foreach_stmt: "for" "each" IDENTIFIER "in" expression block

// for each 带索引变体 (可选)
foreach_with_index_stmt: "for" "each" IDENTIFIER "," IDENTIFIER "in" expression block
```

**示例代码**:
```nexa
flow process_list() {
    items = ["apple", "banana", "cherry"]
    
    // 简单遍历
    for each item in items {
        print(item)
    }
    
    // 带索引遍历
    for each item, index in items {
        print("Index: " + index + ", Item: " + item)
    }
}
```

### 1.3 while 循环

**EBNF 语法设计**:
```ebnf
// while 循环
while_stmt: "while" "(" traditional_condition ")" block
```

**示例代码**:
```nexa
flow countdown() {
    count = 10
    
    while (count > 0) {
        print(count)
        count = count - 1
    }
}
```

### 1.4 break 和 continue

**EBNF 语法设计**:
```ebnf
// break 语句 - 已存在，需确保在 for/while 中可用
break_stmt: "break" ";"

// continue 语句 - 新增
continue_stmt: "continue" ";"
```

## 二、Python 逃生舱设计

### 2.1 定界符选择分析

| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| ` ```python ... ``` ` | 直观 | 破坏 Markdown 渲染 | ❌ 禁止 |
| `python { ... }` | 简洁 | 与字典/集合大括号冲突 | ❌ 禁止 |
| `<<<python ... >>>` | 安全 | 不够直观 | ⚠️ 备选 |
| `<|python|> ... <|end|>` | 安全、醒目 | 较长 | ✅ 推荐 |
| `{{python ... }}` | 类似模板 | 可能与表达式冲突 | ⚠️ 风险 |

**最终选择**: `<|python|> ... <|end|>`

理由:
1. 不与任何主流语言语法冲突
2. 不破坏 Markdown 渲染
3. 醒目易识别
4. 无大括号解析冲突

### 2.2 EBNF 语法设计

```ebnf
// Python 逃生舱语句
python_escape_stmt: "<|python|>" PYTHON_BLOCK "<|end|>"

// Python 代码块 - 原始文本，保留换行
PYTHON_BLOCK: /(?s)(?!<\|end\|>).*/
```

### 2.3 示例代码

```nexa
flow data_processing() {
    // Python 逃生舱 - 导入模块并处理数据
    <|python|>
import pandas as pd
import numpy as np

# 创建示例数据
data = pd.DataFrame({
    'name': ['Alice', 'Bob', 'Charlie'],
    'score': [85, 92, 78]
})

# 计算平均分
avg_score = data['score'].mean()
print(f"Average score: {avg_score}")

# 导出为列表供 Nexa 使用
processed_names = data['name'].tolist()
    <|end|>
    
    // 在 Nexa 中使用 Python 生成的变量
    for each name in processed_names {
        print("Student: " + name)
    }
}
```

## 三、完整 EBNF 扩展

在现有 `flow_stmt` 基础上扩展:

```ebnf
?flow_stmt: assignment_stmt 
          | expr_stmt 
          | semantic_if_stmt      // 语义条件判断
          | traditional_if_stmt   // 新增: 传统 if/else
          | foreach_stmt          // 新增: for each 循环
          | while_stmt            // 新增: while 循环
          | loop_stmt             // 现有: loop...until
          | match_stmt 
          | assert_stmt 
          | try_catch_stmt 
          | print_stmt 
          | break_stmt 
          | continue_stmt         // 新增: continue
          | python_escape_stmt    // 新增: Python 逃生舱
```

## 四、AST 节点设计

### 4.1 传统 if 语句

```python
{
    "type": "TraditionalIfStatement",
    "condition": { ... },           # 逻辑表达式
    "then_block": [ ... ],          # 语句列表
    "else_if_clauses": [            # else if 子句列表
        {
            "condition": { ... },
            "block": [ ... ]
        }
    ],
    "else_block": [ ... ] or None   # else 子句
}
```

### 4.2 for each 语句

```python
{
    "type": "ForEachStatement",
    "iterator": "item",              # 迭代变量名
    "index": "index" or None,        # 索引变量名 (可选)
    "iterable": { ... },             # 可迭代表达式
    "body": [ ... ]                  # 循环体语句
}
```

### 4.3 while 语句

```python
{
    "type": "WhileStatement",
    "condition": { ... },            # 条件表达式
    "body": [ ... ]                  # 循环体语句
}
```

### 4.4 continue 语句

```python
{
    "type": "ContinueStatement"
}
```

### 4.5 Python 逃生舱

```python

```

## 五、代码生成策略

### 5.1 传统 if/else if/else

```python
# Nexa:
if (score >= 90) {
    grade = "A"
} else if (score >= 80) {
    grade = "B"
} else {
    grade = "F"
}

# 生成 Python:
if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
else:
    grade = "F"
```

### 5.2 for each

```python
# Nexa:
for each item, index in items {
    print(item)
}

# 生成 Python:
for index, item in enumerate(items):
    print(item)
```

### 5.3 while

```python
# Nexa:
while (count > 0) {
    print(count)
    count = count - 1
}

# 生成 Python:
while count > 0:
    print(count)
    count = count - 1
```

### 5.4 Python 逃生舱

```python
# Nexa:
<|python|>
import pandas as pd
data = pd.DataFrame({'name': ['Alice']})
names = data['name'].tolist()
<|end|>

# 生成 Python (保持原始缩进，附加当前作用域缩进):
import pandas as pd
data = pd.DataFrame({'name': ['Alice']})
names = data['name'].tolist()
```

## 六、与现有语法的关系

### 6.1 if vs semantic_if 对照表

| 特性 | `if` (新增) | `semantic_if` (现有) |
|------|-------------|---------------------|
| 条件类型 | 确定性布尔表达式 | 自然语言描述 |
| 执行方式 | 直接求值 | LLM 语义判断 |
| 使用场景 | 数据逻辑、状态检查 | 意图识别、内容分类 |
| 性能 | O(1) | 需要 LLM 调用 |

### 6.2 while vs loop...until 对照表

| 特性 | `while` (新增) | `loop...until` (现有) |
|------|---------------|----------------------|
| 终止条件 | 确定性布尔表达式 | 语义条件 |
| 检查时机 | 循环入口 | 循环出口 |
| 使用场景 | 计数、状态迭代 | 内容精炼、迭代优化 |

## 七、实现优先级

1. **P0 - 核心功能**: if/else if/else, for each, while, break, continue
2. **P1 - 逃生舱**: Python escape hatch
3. **P2 - 增强**: for each 带索引变体
